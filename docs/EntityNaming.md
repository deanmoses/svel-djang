# Entity Naming

Pinbase's linkable entities (titles, models, manufacturers, people, etc.) share one canonical name that spans the backend data model, API URLs, and frontend route segments. This doc is the single place that rule lives.

## The rule

Every [`LinkableModel`](../backend/apps/core/models.py) subclass declares an `entity_type` attribute: a **hyphenated singular** string (e.g. `title`, `corporate-entity`, `technology-subgeneration`).

From that one attribute:

- **Backend** uses `entity_type` as-is in URLs like `/api/pages/edit-history/{entity_type}/{slug}/` and in `ChangeSet`/provenance rows.
- **Frontend route segment** is the **plural** form of `entity_type` (e.g. `corporate-entities`, not `corporate-entity` and not `corporateEntities`). Irregular plurals are allowed (`person` → `people`, `series` → `series`).
- **Frontend component props** that take an entity type (e.g. `entityType="gameplay-feature"`) use the singular `entity_type` string, matching the backend.

## Where the canonical list lives

The list is **generated**, not hand-maintained — don't write it into a doc table.

- **Source of truth:** [`apps/core/entity_types.py`](../backend/apps/core/entity_types.py) auto-discovers every `LinkableModel` subclass at import time and exposes `get_linkable_model(entity_type)`.
- **Frontend mirror:** [`frontend/src/lib/api/catalog-meta.ts`](../frontend/src/lib/api/catalog-meta.ts) — regenerated by `python manage.py export_catalog_meta`. Contains `{entity_type, entity_type_plural, label, label_plural}` for every entity. This is the right import for any frontend code that needs to enumerate entity types or look up a plural/label.
- **Parity test:** [`catalog-meta.test.ts`](../frontend/src/lib/api/catalog-meta.test.ts) walks `src/routes/`, requires every `[slug]/` directory to match a `CATALOG_META` entry with `entity_type_plural === directory`, and fails the build if they diverge.

## Adding a new entity type

1. Subclass `LinkableModel` and set `entity_type = "your-thing"` (hyphenated singular).
2. Run `python manage.py export_catalog_meta` to regenerate `catalog-meta.ts`.
3. Create the frontend route directory at `frontend/src/routes/your-things/` (plural form of `entity_type`). The parity test will fail CI if the directory name is wrong.
